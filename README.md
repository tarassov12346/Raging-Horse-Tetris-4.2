# 🧩 Raging Horse Tetris 4.2: Distributed Microservices System

Масштабируемая игровая экосистема, построенная на микросервисной архитектуре. Проект сочетает в себе классический геймплей с продвинутыми техниками авторизации, распределенного хранения данных и визуализации игровых состояний.

## 🏗 Архитектурная логика (Data Flow)

1.  **Identity & Access**: Пользователь начинает с `USERS-microservice`, где проходит регистрацию/аутентификацию и получает JWT-токен.
2.  **Smart Routing**: `Gateway-microservice` проверяет валидность токена и транслирует данные пользователя (ID и Username) непосредственно в игровое приложение.
3.  **Polyglot Persistence**:
    *   **PostgreSQL (`PG-microservice`)**: Отвечает за транзакционные данные (лучшие игроки, статистика сессий, личные рекорды).
    *   **MongoDB (`Mongo-microservice`)**: Хранит тяжелый контент и гибкие состояния (аватары, дампы прерванных игр для продолжения, скриншоты финальных моментов).
4.  **Hybrid Communication**:
    *   **gRPC (Protobuf)**: Используется для высокоскоростной передачи игровых данных, аватаров и скриншотов.
    *   **OpenFeign**: Используется для простых административных и CRUD операций (удаление и т.д.).

## 🌟 Ключевые технологические решения
*   **Visual Snapshots (Playwright)**: Уникальный механизм захвата финальных моментов игры через headless-браузер прямо в Docker-контейнере. Снимки сохраняются в MongoDB и отображаются в профиле как "последняя" и "лучшая" игра.
*   **Save & Resume**: Возможность полной сериализации состояния игры в MongoDB с последующим восстановлением сессии.
*   **Modern Java**: Использование **Java 21** и **Spring Boot 3.4.2** с полной поддержкой **Jakarta EE**.

## 🛠 Технологический стек
- **Core:** Java 21, Spring Boot 3.4, Spring Cloud (Eureka, Gateway, OpenFeign).
- **Security:** JWT (JSON Web Tokens).
- **Communication:** gRPC, WebSockets, REST.
- **Databases:** PostgreSQL, MongoDB.
- **In-Memory:** Hazelcast.
- **DevOps:** Docker, Playwright (Headless rendering).

## 📊 Схема микросервисов
* **Service Discovery (Eureka-microservice)**: Центральный узел регистрации всех сервисов. Обеспечивает динамическое обнаружение микросервисов, что позволяет системе масштабироваться без жесткой привязки IP-адресов в конфигурации.

- `USERS-microservice`: Центр авторизации и выдачи токенов.
- `Gateway-microservice`: Единый вход, фильтрация трафика и проверка безопасности.
- `PG-microservice`: Работа с реляционными данными (статистика, топ-игроков).
- `Mongo-microservice`: Хранилище бинарных данных (фото, аватары) и состояний игр.
- `Raging-Horse-Tetris-4.2`: Основной игровой движок, интегрирующий данные из всех сервисов.
